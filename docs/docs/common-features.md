These examples show common tasks using the framework features that exist today.

## Shared navigation

Put shared navigation in a template so links can be generated by your route handlers.

```html
{{ define "pages/home" }}
  <nav>
    <a href="{{ .HomeURL }}">Home</a>
    <a href="{{ .ProjectsURL }}">Projects</a>
  </nav>
  <h1>{{ .Title }}</h1>
{{ end }}
```

```go
return ctx.Render("Home", "pages/home", map[string]string{
	"Title":       "Home",
	"HomeURL":     ctx.MustURL("home", nil),
	"ProjectsURL": ctx.MustURL("projects.index", nil),
})
```

## Server-rendered data

```go
app.Get("/projects", func(ctx *gofast.Context) gofast.Page {
	projects := loadProjects(ctx.RequestContext())

	return ctx.Render("Projects", "pages/projects", map[string]any{
		"Projects": projects,
	})
}).Name("projects.index")
```

Keep data loading, authorization, and rendering decisions in Go. Templates should display prepared data, not contain application logic.

## Custom status codes

```go
return gofast.Page{
	Title:  "Created",
	Body:   gofast.HTML("<h1>Created</h1>"),
	Status: 201,
}
```
